# Project Structure
---

To keep your Brick Breaker prototype organized, I recommend using a structure like in this chart:

```plaintext
res://
├── scenes/
│   ├── Paddle.tscn
│   ├── Ball.tscn
│   ├── Brick.tscn
│   └── Level.tscn
├── scripts/
├── assets/
└── main.tscn
```


---

## Why this structure?

- Scenes remain reusable  
- Code stays modular and easy to debug  
- Level logic stays separate from object logic
